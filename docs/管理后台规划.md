# 星宝数据平台 — 管理后台规划

> 记录时间：2026-05-27

## 四期总体规划

| 期次 | 模块 | 内容 | 估算 |
|------|------|------|------|
| **第一期** | 用户管理 + 反馈事件 + 总览 | 用户 CRUD、登录日志、反馈事件管理（详情页+批量操作）、admin 总览趋势 | 2天 |
| **第二期** | 数据源管理 + 缓存管理 | 数据上传/切换/状态看板、缓存统计/清除 | 2天 |
| **第三期** | 模板管理 + 系统配置 + 重启 | 模板在线编辑/启禁、.env 参数管理、一键重启 | 1-2天 |
| **第四期** | 监控统计 + 查询排行 + 告警 | API 调用统计、用户查询排名、异常告警时间线 | 2天 |

## 第一期详细方案

### 模块一：用户管理

**权限分级**：admin / user 两级，不引入更细权限

**后端接口**（admin_router.py，前缀 `/api/admin`）：

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/admin/users | 用户列表（分页+搜索+状态筛选） |
| POST | /api/admin/users | 新增用户 |
| PUT | /api/admin/users/{username} | 编辑用户（角色/状态/展示名/备注） |
| DELETE | /api/admin/users/{username} | 删除用户 |
| POST | /api/admin/users/{username}/reset-password | 重置为 .env 默认密码 |
| POST | /api/admin/users/{username}/toggle-status | 启用/禁用 |
| GET | /api/admin/users/{username}/history | 某用户查询历史 |
| GET | /api/admin/users/stats | 用户使用统计 |

**前端组件**：AdminUsers.tsx
- 用户表格：用户名、角色、状态、创建时间、最后登录、查询次数
- 新增/编辑 Modal
- 操作：重置密码、禁用/启用、删除（确认）
- 搜索+筛选

### 模块二：反馈事件管理

**粒度**：完整详情页 + 批量操作

**后端接口**：

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/admin/incidents | 事件列表（分页+类型+状态+时间筛选） |
| GET | /api/admin/incidents/stats | 事件统计 |
| GET | /api/admin/incidents/{id} | 事件详情 |
| PUT | /api/admin/incidents/{id}/status | 变更状态（resolved/wontfix/reopen） |
| POST | /api/admin/incidents/{id}/analyze | 触发 AI 重新分析根因 |
| POST | /api/admin/incidents/batch-status | 批量变更状态 |
| POST | /api/admin/incidents/batch-delete | 批量删除 |
| GET | /api/admin/incidents/export | 导出 CSV |
| DELETE | /api/admin/incidents/{id} | 删除单条 |

**前端组件**：AdminIncidents.tsx
- 筛选栏：类型/状态/环境/时间范围/关键词
- 事件卡片列表：ID、类型标签、状态标签、问题摘要、时间
- 点击展开详情：完整问题、SQL、根因分析、修复建议
- 批量操作：多选→统一标记已处理/忽略/删除

### 模块三：Admin 总览

**后端接口**：

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/admin/dashboard | 总览指标（用户数、事件数、查询量趋势） |

**前端组件**：AdminOverview.tsx
- 核心指标卡：用户总数、今日活跃用户、待处理事件数、总查询次数
- 近7日查询量趋势图（ECharts 折线图）
- 待处理事件快捷入口

### 新增数据库表

在 star-query-history.db 中新增：

```sql
-- 用户管理
CREATE TABLE users (
    username TEXT PRIMARY KEY,
    role TEXT NOT NULL DEFAULT 'user',       -- admin / user
    status TEXT NOT NULL DEFAULT 'active',   -- active / disabled
    display_name TEXT DEFAULT '',
    created_at TEXT NOT NULL,
    last_login_at TEXT,
    note TEXT DEFAULT ''
);

-- 反馈事件（替代 feedback_review/ JSON 文件系统）
CREATE TABLE incidents (
    id TEXT PRIMARY KEY,
    type TEXT NOT NULL,                      -- validation_fail / user_dislike
    env TEXT DEFAULT 'prod',
    status TEXT NOT NULL DEFAULT 'pending',  -- pending / analyzing / resolved / wontfix
    question TEXT NOT NULL,
    sql TEXT DEFAULT '',
    error TEXT DEFAULT '',
    warnings TEXT DEFAULT '[]',
    intent_info TEXT DEFAULT '{}',
    history_id TEXT DEFAULT '',
    feedback_comment TEXT DEFAULT '',
    root_cause TEXT DEFAULT '',
    fix_proposal TEXT DEFAULT '',
    resolved_at TEXT,
    resolver TEXT DEFAULT '',
    created_at TEXT NOT NULL
);

-- 操作审计日志
CREATE TABLE operation_logs (
    id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    action TEXT NOT NULL,
    target TEXT DEFAULT '',
    detail TEXT DEFAULT '',
    ip_address TEXT DEFAULT '',
    created_at TEXT NOT NULL
);
```
